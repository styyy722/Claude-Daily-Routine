# Claude Daily Finance Routine Instructions

This repository stores automated daily financial news reports generated by Claude Routine.

## Main task

Each routine run must generate **two** new markdown reports for the **previous calendar day** using Australia/Sydney time (AEST/AEDT): one in English and one in Chinese (Simplified).

**Date calculation (do this first):**
1. Get today's date in Australia/Sydney timezone.
2. Subtract one day — that is the **target date** for this run.
3. The target filenames are:
   - English: `outputs/daily-finance/YYYY-MM-DD-financial-news-report.md`
   - Chinese: `outputs/daily-finance/YYYY-MM-DD-financial-news-report-zh.md`
4. For each file, check whether it already exists:
   - If it does **not** exist: generate and save it.
   - If it **already exists**: skip generation for that language and use the existing file.

Do **not** re-use or re-send any report from a date other than the target date.

## Saving and committing to GitHub

After generating both report files, commit and push them to the GitHub repository:

```
git add outputs/daily-finance/YYYY-MM-DD-financial-news-report.md outputs/daily-finance/YYYY-MM-DD-financial-news-report-zh.md
git commit -m "Add daily financial news reports for YYYY-MM-DD"
git push origin main
```

- Only add the two report files for the target date — do not add any other files.
- If a report already existed and was not regenerated, skip adding that file to the commit.
- If both files already existed, skip the commit entirely.

## Report rules

- English report: use the template in `templates/daily-financial-report-template.md`.
- Chinese report: use the template in `templates/daily-financial-report-template-zh.md`.
- Keep each report around one page.
- The Chinese report is a full translation and localisation of the English report — same data, same structure, written naturally in Simplified Chinese.
- Include source links where available.
- Do not invent market data.
- Clearly state when data is unavailable.
- Do not provide personalised financial advice.
- Save only the final report files.

## Suggested public sources

Use reliable public sources to research the target date's market activity:
- Reuters business and markets coverage
- Yahoo Finance / market data pages
- MarketWatch
- CNBC Markets
- Federal Reserve official releases
- Reserve Bank of Australia releases
- ASX market information
- Commodity market pages for gold, oil, and iron ore

Prioritise market-moving information over general news.

## Discord delivery

After saving (or confirming) both reports, send both the English and Chinese full report content as Discord messages using the environment variables:

- DISCORD_BOT_TOKEN
- DISCORD_CHANNEL_ID

Send in this order:
1. Full English report
2. Full Chinese report

Discord messages have a 2000-character limit. Split each report into multiple sequential messages at logical section boundaries (e.g. after the Executive Summary, after the Market Snapshot, etc.) if needed. Do not use file attachments.

Do not include any Discord token or secret in this repository.
